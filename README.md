# Складський облік — PHP/MariaDB MVC
Веб-додаток для кількісного обліку руху матеріалів на складах з автоматичним розрахунком списання за витратою ресурсів.

### Форма витрати ресурсів — UX та валідація
- Стрілочка → на рухи не показується якщо дельта = 0 (списання не відбулось)
- Зарезервовано місце під контекстний блок у формі — форма не скролиться при появі підказок
- Помилка «не менше / не більше» показується тільки під датою в contextInfo, без дублю зверху модалки
- Клієнтська валідація перед відправкою: reading >= prev і reading <= next

### Каскадне перепроведення та валідація показників
- При додаванні витрати ресурсу «заднім числом» — автоматично перепроводяться всі пізніші записи
- При редагуванні — каскадне перепроведення всього ланцюжка
- При видаленні — перепроведення наступних записів
- Дозволено delta = 0 (склад не працював, перенесення дати розкидки)
- Серверна перевірка: показник не менше попереднього і не більше наступного
- API `/resources/context` — повертає prev/next для дати з підтримкою exclude_id

### Контекст при вводі витрати ресурсів
- Порядок полів: Дата → Склад → Ресурс → запит контексту → Показник
- Після вибору дати+складу+ресурсу — автозапит до бази
- Показує попередній показник з датою та наступний показник з обмеженням
- При зміні дати — контекст перезавантажується

### Звіти — стиль та running balance
- Прибрано зелений/червоний колір з колонок Прихід/Витрата — всі числа чорні
- Підсумкові рядки (по складу/матеріалу та Разом) — жирні
- Рядки деталізації (рух) — не жирні
- У деталізації показується кінцеве сальдо після кожного руху
- Від'ємне сальдо виводиться червоним
- Деталізація рухів у зворотному порядку дат (новіші зверху)

### Карточка перегляду руху
- Прибрано «(автоматичний)» із заголовка — тепер просто «Перегляд руху»
- Для розрахункових рухів додані рядки: Норма, Ресурс, Показник
- Друк руху: примітка не обрізається, колонка Дії не друкується

### Примітка руху — норма зафіксована
- Формат: `Розр. (150 км, Нр 0.35), коментар`
- Норма записується в момент списання для аудиту
- При розносці по днях — в примітці кожного дня стоїть денна дельта, а не загальна

### Автоматичні записи в таблиці Рух
- Для автоматичних записів у закритому періоді: 🔒 + → (обидва значки)
- Даблклік по автоматичному рядку → картка перегляду (read-only)
- Даблклік по закритому ручному рядку → теж картка перегляду
- Іконка → замість ⚙️ для переходу до витрати ресурсів

### Розноска витрати ресурсу по днях
- Галочка «Рознести по днях» у довіднику норм
- Витрата ділиться рівномірно по кожному дню між попереднім і поточним показником
- Заокруглення: hm → до 10 хвилин, int → до цілих, dec2 → до сотих
- Різниця від заокруглень — в останній день
- Кожен день = окремий запис у movements з resource_log_id

### Логіка списання ресурсу
- Списуємо З складу (для якого норма), відносимо НА склад з норми
- Перейменовано «Списати зі складу» → «Віднести на склад»
- Примітка з витрати ресурсу додається до примітки руху через кому
- Перший показник — нічого не списується, тільки фіксується

### Формат вводу ресурсу
- Нове поле `format` в типах ресурсів: int, dec2, hm
- int: заборонені крапка і кома при вводі
- dec2: до сотих
- hm: ввід як год:хв, конвертація в десяткове перед збереженням
- Серверна перевірка формату

### Стрілка → у витраті ресурсів
- Для кожного рядка з дельтою > 0 — стрілка → на відфільтрований рух
- Фільтр: по складу + діапазон дат від попереднього до поточного списання

### Ресурсні поля в movements
- Нові колонки: resource_value, resource_delta, resource_rate
- Заповнюються при автоматичному створенні рухів
- Використовуються в експорті та картці перегляду

### Імпорт рухів — захист автоматичних записів
- «Видалити дані перед імпортом» — видаляє тільки ручні рухи
- Автоматичні (resource_log_id IS NOT NULL) залишаються
- Експорт: додані колонки Показник, Дельта, Норма для автоматичних рухів

### Витрата ресурсів — система автоматичного списання
- Таблиці: resource_types, warehouse_resources, resource_rates, resource_logs
- Типи ресурсів (Пробіг/км, Мотогодини/год) з форматом вводу
- Прив'язка ресурсів до складів
- Норми списання: матеріал + норма на 1 од. ресурсу + куди віднести
- Журнал показників: ввід одометра/таймера → авторозрахунок дельти → створення рухів
- Друк та експорт витрати ресурсів
- Навігація: «Витрата ресурсів» у розділі Документи, «Норми» та «Типи ресурсів» у Ресурси

### Закритий період
- Таблиця config (key-value) з датою закритого періоду
- Рухи до дати закриття заблоковані від модифікації (🔒)
- Серверна перевірка при save/delete/import
- Імпорт з урахуванням: рядки у закритому періоді пропускаються, видаляються тільки після
- Панель керування: у dropdown поруч з глобальним діапазоном дат

### Кнопка «Додати» в шапці таблиці
- Перенесена з page-header у заголовок колонки «Дії»
- Довідники, рух, витрата ресурсів, норми

### Подвійний клік по рядку → редагування
- Довідники складів/матеріалів: завжди
- Рух: тільки для відкритих ручних рухів
- Витрата ресурсів: тільки для відкритих записів

### Кнопки фільтрів у одному рядку
- «Застосувати» / «Скинути» / «Сформувати» перенесені в рядок з фільтрами
- Рух, звіт по складу, звіт по матеріалу, витрата ресурсів

### Висота сторінок під Nextcloud iframe
- CSS-змінна --nc-header: 50px
- .app: height: calc(100vh - var(--nc-header))
- Таблиці скролються всередині, сторінка не має вертикального скролу
- card-stretch + table-scroll з flex-grow
- Підсумкова інформація (card-footer-info) всередині карточки

### Прибрано верхню шапку
- Синій header «Складський облік» видалено
- Глобальний діапазон дат перенесено на сторінки (рух, звіти)
- Sidebar починається з верху сторінки

### Збереження діапазону дат у cookie
- Cookie sklad_date_from / sklad_date_to (365 днів)
- SameSite=None для роботи в iframe Nextcloud
- Fallback: cookie → session → default

### Сторінка за замовчуванням
- При відкритті `/sklad/` відкривається Рух матеріалів (замість довідника складів)

### Сортування таблиці руху
- Клікабельні заголовки з індикаторами ▲▼⇅
- За замовчуванням: дата спадання (останні вгорі)
- Сортування зберігається при застосуванні фільтрів

### Sticky шапка таблиць
- table-scroll з overflow і sticky header
- Працює в довідниках, русі, звітах, витраті ресурсів

### Фільтр «Прихід ззовні» та «Списання»
- Спецпункти ⬇ / ⬆ у фільтрі складу на сторінці руху
- Серверна обробка: warehouse_from_id IS NULL / warehouse_to_id IS NULL

### Захист довідників
- Якщо склад/матеріал використовується в русі — кнопка «Видалити» не показується
- Перевірка через getUsedIds() одним запитом

### Автостворення при імпорті
- Якщо складу/матеріалу немає в довіднику — створюється автоматично (findOrCreate)
- Галочка «Видалити поточні дані руху перед імпортом»

### Дата у форматі dd.mm.yyyy
- Всі видимі дати: таблиці, звіти, деталі, експорт, підказки імпорту
- input type="date" та БД залишаються в ISO форматі

### Друк руху
- Кнопка «Друк» поруч з «Експорт»
- print-header з фільтрами, сортуванням, кількістю записів, датою друку
- Примітка не обрізується, колонка Дії не друкується

### Підсвічування рядка при переході зі звіту
- Жовтий фон + синя смужка зліва
- Тримається 15 секунд або до кліку на іншу строчку
- Плавне зникнення

### Експорт руху в Excel
- Формат .xlsx без зовнішніх бібліотек (ZipArchive + XML)
- Колонки: Дата, Звідки, Куди, Матеріал, Кількість, Показник, Дельта, Норма, Примітка
- Автофільтр на заголовку

### Імпорт руху з Excel
- Парсинг .xlsx (shared strings, inline strings)
- Підтримка форматів дати: DD.MM.YYYY, YYYY-MM-DD, DD/MM/YYYY, Excel serial
- Зіставлення назв без урахування регістру

### Множинний вибір складів у звіті по матеріалу
- Модалка з чеклістом + пошук
- Кнопки «Обрати все» / «Зняти все»

### Autocomplete select
- Компонент buildAutocomplete для всіх <select class="autocomplete">
- Пошук по назві, Enter для вибору першого видимого
- data-submit-on-change для автоматичної відправки форми
- z-index для коректного відображення поверх таблиць

### Модальні вікна замість окремих сторінок
- Форми складів, матеріалів, рухів відкриваються popup-ом поверх таблиці
- AJAX збереження без перезавантаження
- Searchable select у модалках

### MVC архітектура
- index.php — роутер з автозавантаженням класів
- core/Controller.php — render, redirect, flash, get, post, json
- core/Model.php — getAll, getById, delete
- Контролери: Warehouses, Materials, Movements, Reports, Settings, Resources
- Моделі: Warehouse, Material, Movement, Config, Resource
- Views: layout.php + окремі шаблони

### База даних MariaDB
- warehouses, materials, movements, config
- resource_types, warehouse_resources, resource_rates, resource_logs
- Зовнішні ключі, індекси, CHECK constraints
